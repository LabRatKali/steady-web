Privacy policy

Plain language for parents and for people using Steady on their own phone. Steady asks for strong phone permissions — you deserve a clear answer about data.

Last updated: July 12, 2026 · Questions: labratcomputers@gmail.com

Short version

  • No ads. No analytics SDK. No selling your data.
  • Most settings, PIN/password hashes, app timers, and Steady Face stay on the phone.
  • Family features (pairing, rules, Approvals, to-dos) sync only when you set them up — through a private GitHub sync repo, not a Steady-owned “cloud account” product.
  • Optional crash reports can upload error text to that same sync path so bugs can be fixed.
  • DNS filtering (when on) asks Cloudflare for Families to resolve allowed names — not a Steady tracking server.

Who this is for

Steady is an Android anti-addiction app for self-focus and for parent / child setups. It uses Device Admin, Accessibility, Usage access, and (if you choose Steady Face) the camera — so you can lock profiles, block apps, and keep a day schedule. Those permissions are for enforcement on the device, not for building an advertising profile.

What stays on the device

  • Unlock codes (PIN, pattern, passphrase, and similar) — stored as local secrets / hashes, not sent for “login to Steady cloud.”
  • Steady Face — face landmarks become a local template on the phone. Face photos / templates are not uploaded for family sync.
  • Daily usage minutes, per-app use, Accessibility events, and Usage Access readings used to enforce blocks — processed on-device.
  • Local chores, categories, appearance settings, and most preferences.

What can leave the device (only if you use those features)

Family pairing & rules

When you create a family code and pair phones, Steady syncs family policy so parent and child stay aligned. That can include: budgets / profile minutes, bedtime, school hours, weekend Fun bonus, day schedule, web-filter toggles, age band, cool-downs, homework / to-do gates, install-approval flags, and related family settings. Pairing records include device IDs and a family secret used for that link.

Approvals & to-dos

“Ask for Fun,” remote Approve / Deny, family to-do lists (titles, status, reward minutes), and optional install-approval requests can sync between the paired phones so the parent inbox works when you are not standing next to each other.

Nearby Approve

If you use Nearby (Bluetooth / local wireless) Approve, short approval messages can travel phone-to-phone over Google’s Nearby Connections — not through an ad network.

Optional location features

If geofence / zone features are enabled, zone definitions (which may include coordinates) and enter/exit alerts (zone name / event — not a continuous live map feed to advertisers) can sync as part of family features.

Updates

Steady may check GitHub for a newer APK (release metadata / download). That is an update check, not usage tracking of which apps your child opened today.

Crash reports

If the app crashes, it may store an error report on the phone and later upload it to the sync path (device id, app version, message, stack trace, timestamp) so the developer can fix bugs. This is not a marketing analytics package.

Web filter / DNS

When the family web filter is on, Steady runs a local DNS path and forwards allowed lookups to Cloudflare for Families (age-aware adult / malware filtering). Blocked names can be answered locally. This is filtering — not Steady selling browse history.

Where synced family data goes

Steady does not run a separate “Steady cloud account” product with a developer-owned app server in the usual SaaS sense. Family sync, update checks, and optional crash uploads use GitHub repositories operated for Steady (sync + public downloads). Synced payloads are obfuscated on the app side before storage. Access to that private sync material is controlled by the project’s GitHub credentials — not “sold to advertisers.”

Self-focus users who never pair a family may never send family policy or Approve traffic at all.

What we don’t do

  • No advertising SDK, no AdMob, no selling personal data.
  • No Firebase Analytics / Crashlytics-style product analytics package in the app dependencies.
  • We don’t upload your Steady Face template or raw camera stream for family sync.
  • We don’t upload Accessibility event streams or Usage Stats histories as a feed for third parties.

Children & parents

On a child phone, Steady is meant for a parent (or guardian) to set boundaries. Family sync exists so Approvals and rules can work across two phones. If you are a parent: you choose the family code, who pairs, and which features you turn on. If you stop using family features, you can stop pairing / stop using those sync paths; local enforcement settings remain on the device until you change or uninstall them (uninstall may require turning protection off first — see the setup guide).

This website

The public site is hosted for downloads and setup help. The “Leave an idea” box opens your own email app — it does not upload the text to a Steady web form server. The download page may ask GitHub’s public API for the latest release name so the version line stays current.
